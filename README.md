## üì• Getting Started

### Prerequisites
- A web browser (Chrome, Firefox, Safari, Edge)
- Git (for cloning the repository)
- Optional: Node.js (if running a local server)

### Installation & Local Setup

#### **Option 1: Clone and Open Directly**
Fastest way to run locally:

```bash
# Clone the repository
git clone https://github.com/Lerato-leo/Personal-portfolio.git

# Navigate to the project directory
cd Personal-portfolio

# Open in your default browser
# On Windows
start index.html

# On macOS
open index.html

# On Linux
xdg-open index.html
```

#### **Option 2: Using a Local Server (Recommended)**

**With Python:**
```bash
# Clone the repository
git clone https://github.com/Lerato-leo/Personal-portfolio.git
cd Personal-portfolio

# Python 3.x
python -m http.server 8000

# Python 2.x
python -m SimpleHTTPServer 8000

# Visit: http://localhost:8000
```

**With Node.js (Live Server):**
```bash
# Install live-server globally
npm install -g live-server

# Clone and navigate
git clone https://github.com/Lerato-leo/Personal-portfolio.git
cd Personal-portfolio

# Start server
live-server

# Automatically opens http://localhost:8080
```

**With Node.js (HTTP Server):**
```bash
# Clone the repository
git clone https://github.com/Lerato-leo/Personal-portfolio.git
cd Personal-portfolio

# Install http-server
npm install -g http-server

# Start server
http-server

# Visit: http://localhost:8080
```

## üöÄ Deployment

### Deployed on Vercel

This portfolio is automatically deployed on **Vercel** for fast, reliable hosting with zero-config deployment.

#### **Deployment Architecture:**
- **Platform:** Vercel (https://vercel.com)
- **Auto-Deploy:** Connected to GitHub main branch
- **CDN:** Global edge network for optimal performance
- **SSL/HTTPS:** Automatic certificate management
- **Custom Domain:** (Optional) Can be configured

#### **Live URL:** 
üåê [https://lerato-portfolio.vercel.app](https://lerato-portfolio.vercel.app)

#### **How It's Deployed:**

1. **GitHub Connection** - Repository connected to Vercel
2. **Automatic Builds** - Every push to `main` branch triggers a deployment
3. **Build Process** - Vercel serves static files (HTML, CSS, JS, images)
4. **Instant Preview** - Staging URL generated for each push
5. **Production Deployment** - Merged to main = live update

#### **Deploy Your Own Copy (Fork to Vercel):**

```bash
# 1. Fork the repository to your GitHub account
# Visit: https://github.com/Lerato-leo/Personal-portfolio
# Click "Fork" button

# 2. Go to Vercel
# Visit: https://vercel.com

# 3. Click "New Project"
# Select your forked repository
# Click "Import"

# 4. Vercel auto-detects it's a static site
# Click "Deploy"

# Your portfolio is now live!
# Vercel provides you a URL like: https://your-portfolio.vercel.app
```
